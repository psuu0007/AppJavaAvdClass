함수(function)
일련의 실행문들을 메모리에 저장했다가 필요할 때 해당 함수의 이름으로
다시 호출하여 사용할 수 있다.



이벤트(event)
웹 브라우저에서 일어나는 모든 행동을을 이벤트(event)라고 한다
특정 이벤트가 발생하는 시점에서 함수를 호출하거나 
스크립트를 실행하게 하는 요소를 이벤트 핸들러 라고 부른다.

자바스크립트는 브라우저가 생성한 DOM 구조를 통해 각 태그와의 상호작용을 가능하게 함으로써
웹 페이지가 보다 인터렉티브(다이나믹)하게 작동하도록 돕는 것이다.


변수의 라이프 사이클
지역변수(Local Variable)와 전역변수(Global Variable)

함수 내에서의 변수는 지역변수와 전역변수로 구분된다.
지역변수란?
변수가 정의된 함수 내에서만 사용이 가능한 변수이며,

전역변수란?
현재 페이지 내 자바스크립트 어디서든 사용이 가능한 변수이다.

Q&A
지역변수와 전역변수를 구분하는 이유는 멀까?
변수의 중복 사용을 막기 위해서다. 전역변수와 지역변수의 이름이 같다는 것만으로
오류가 발생하지는 않으며, 해당 함수 내에서는 지역변수가 우선 호출된다.
하지만 프로그램이 복잡해질수록 깔끔한 실행을 위해서는 될수록 구분해주는 것이 좋겠죠? ^^

기존 방식(DOM Level0)이벤트의 문제점

객체에 이벤트를 직접 연결하여 작동하게 하는 고전적인 방식을 DOM level0 이라고 한다.
객체 하나와 이벤트 하나를 연결하여 작동하는 이 방식은 한 객체에 동시에 같은 이벤트를 
두번 이상 적용할 수 없다는 문제를 안고 있다. 한 객체에 같은 이벤트를 두 번 이상 적용하게 
되면 제일 마지막에 적용된 이벤트만 실행된다.


DOM Level2
한 객체에 같은 이벤트를 두 번 적용할 수 없는 이전 방식의 불편함을 해결하기 위해
개발된 방식이다.
하지만 W3C(국제 웹 컨소시엄)의 웹 표준안과 다른 독자적 구현 방식을 가진
마이크로소프트의 익스플로러(IE)가 가장 많은 사용자를 지닌 사실상의 표준 웹브라우저로(우리나라기준)
지위를 갖게 되면서, 정확한 이벤트 구현을 위한 이벤트 등록 함수가 서로 달라진다는 불편함이 생겼다. 

결국 이 level2 방식으로 구현한 이벤트를 모든 웹 사용자에게 정확하게 보이려면
표준을 따르는 크롬, 파이어폭스, 사파리 등이나 비표준 방식으로 작동하는 ie 버전의 브라우저 등에서
모두 정상 동작하도록 구분자(if/else if)를 나눠 작성해야 한다.

표준과 비표준을 구분하는 이벤트 등록은 다음과 같다
if(window.addEventListener){    크롬등
	표준 이벤트 등록 메서드;
}else if(window.attachEvent){  각 ie 8버전 이하
	비표준 이벤트 등록 메서드;
} 

결국 사용자의 브라우저가 표준이면 if문
비표준이면 else if문을 만족하도록 작성해왔다.
